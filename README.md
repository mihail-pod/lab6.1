# Компилятор арифметических выражений в префиксной форме

Учебный проект для лабораторной работы по дисциплине «Разработка компиляторов и интерпретаторов».

Проект реализует простой конвейер компиляции без ANTLR, генераторов парсеров и сторонних parsing-библиотек:

- лексический анализ;
- рекурсивный синтаксический анализ;
- построение AST;
- оптимизацию AST;
- генерацию таблицы четверок;
- генерацию TASM-подобного assembler-кода;
- консольный режим;
- Swing GUI;
- JUnit 5 тесты.

## Технологии

- Java 17+
- Maven
- Swing
- JUnit 5

## Архитектура

- `lexer` - собственный лексер. Типы токенов находятся во вложенном enum `Token.TokenType`.
- `parser` - рекурсивный descent parser. Строит AST по префиксной грамматике.
- `ast` - дерево выражения: `Node`, `BinaryOperationNode`, `ValueNode`.
- `optimizer` - простые оптимизации дерева.
- `quadruples` - генерация промежуточного представления `(op, arg1, arg2, result)`.
- `assembler` - генерация TASM-подобного assembler-кода в сегментном стиле.
- `gui` - Swing-интерфейс.
- `utils` - `CompilerService`, результат компиляции и общий `CompilerException`.
- `tests` - JUnit-тесты.

## Формат входных данных

Поддерживается префиксная запись:

```text
(+A,B)
(*(+A,B),(-C,2))
(/(*(-A,B),(-C,2)),(+C,D))
```

Операции:

```text
+ - * /
```

Операнды:

- целые числа;
- переменные из одного символа.

## Оптимизации

```text
X + 0 -> X
X - 0 -> X
X * 1 -> X
X * 0 -> 0
X / 1 -> X
```

Если правый операнд деления является числом `0`, компилятор записывает ошибку в `errors.log`.

## Пример четверок

Для входа:

```text
(*(+A,B),(-C,2))
```

получится:

```text
(+, A, B, T1)
(-, C, 2, T2)
(*, T1, T2, T3)
```

## Пример assembler

Файл `output.asm` создается в сегментном стиле, близком к классическому TASM/DOS:

```asm
stack1 segment
    db 256 dup(?)
stack1 ends

data segment
    A dw 0
    B dw 0
    T1 dw 0
data ends

code segment

start:
    assume ss:stack1, ds:data, cs:code

    mov ax, data
    mov ds, ax

    ; (+, A, B, T1)
    mov ax, A
    add ax, B
    mov T1, ax

    mov ah, 4ch
    int 21h

code ends
end start
```

## Обработка ошибок

Для всех этапов используется один общий класс `CompilerException`.
Он применяется в лексере, парсере, оптимизаторе и генераторе assembler-кода.

Ошибки записываются в `errors.log`.

## Консольный запуск

1. Запишите выражение в `input.txt`.
2. Выполните:

```bash
mvn clean package
mvn exec:java -Dexec.mainClass=Main
```

После запуска создаются файлы:

- `quadruples.txt`
- `output.asm`
- `errors.log`

## Запуск GUI

Из IDE запустите класс:

```text
gui.CompilerGui
```

В окне можно выбрать файл, изменить выражение вручную и нажать `Компилировать`.
GUI показывает AST, оптимизированный AST, четверки, assembler и ошибки.

## Запуск тестов

```bash
mvn test
```

Тесты покрывают:

- лексер;
- парсер;
- оптимизатор;
- генератор четверок;
- корректные и ошибочные входные данные.

## Выходные файлы

- `quadruples.txt` - таблица четверок.
- `output.asm` - TASM-подобный assembler-код.
- `errors.log` - ошибки лексера, парсера, оптимизатора, генератора assembler или работы с файлами.
